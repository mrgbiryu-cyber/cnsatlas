# 3page Baseline Checklist

## 목적

`3page`는 `1page`, `2page`처럼 화살표 몇 개만 고치면 되는 수준이 아니다.
복합 UI 전체를 `read-first` 기준으로 다시 봐야 하므로, 먼저 비교 기준선을 고정한다.

기준 비교 대상:

- 원본 PPT `slide 29`
- 고급 플러그인 결과 `Page 3`
- 현재 우리 변환기 결과

핵심 질문:

> 현재 결과가 고급 플러그인 수준에 근접하거나, 최소한 PPT 사용자가 해석 가능한 수준인가?

## 1. 상단 마스터 / 헤더 영역

확인 항목:

- 상단 메타 정보가 한 구역으로 읽히는가
- 좌측 정렬 붕괴 없이 정상적인 레이아웃을 유지하는가
- 제목, 버전, 보조 정보가 서로 겹치거나 한곳에 몰리지 않는가

실패 기준:

- 상단 텍스트가 좌측에 몰림
- 한 줄 또는 한 블록처럼 뭉쳐 보임
- 헤더와 본문이 구분되지 않음

## 2. 긴 텍스트 / 설명 박스

확인 항목:

- 긴 문장이 박스를 벗어나지 않는가
- 문단 구분과 줄바꿈이 의미를 해치지 않는가
- 중첩 박스 내부 텍스트가 읽기 쉬운가

대표 확인 예:

- `V 5.0 [BTOCSITE-88928] ...` 형태의 긴 문구

실패 기준:

- 줄바꿈이 전혀 되지 않음
- 박스 밖으로 넘침
- 텍스트가 겹침 또는 잘림

## 3. 표 가독성

확인 항목:

- 셀 높이와 폭이 읽을 수 있는 수준인가
- 디스크립션이 한 줄로 눌려 찌그러지지 않는가
- 표 상단으로 모든 셀이 몰리지 않는가
- merge/설명 셀 구조가 읽히는가

실패 기준:

- 셀이 위로 몰림
- 표 설명이 보이지 않음
- 경계와 내용 관계가 흐려짐

## 4. 옵션 / 가격 / 이미지 블록

확인 항목:

- 옵션 선택 영역이 한눈에 읽히는가
- 가격/할인/설명/이미지 관계가 유지되는가
- 관련 정보가 서로 멀리 떨어지지 않는가

실패 기준:

- 가격/옵션/설명 블록이 분산돼 보임
- 이미지와 텍스트 관계가 끊김
- 사용자가 흐름을 한 번에 읽기 어려움

## 5. 그룹 / 섹션 구조

확인 항목:

- 동일한 의미의 블록들이 시각적으로 묶여 보이는가
- 그룹이 납작하게 풀려서 화면이 산만해지지 않는가
- 박스 / 카드 / 그룹 단위의 정보 계층이 유지되는가

실패 기준:

- 그룹이 해체되어 관계가 흐려짐
- 섹션 구분이 사라짐

## 6. 고급 플러그인 대비 비교 축

비교 축:

- visual readability
- grouping fidelity
- text containment
- table readability
- information hierarchy

질문:

- 고급 플러그인과 비교했을 때 “심하게 떨어진다”는 느낌이 남는가
- 우리가 구조는 더 좋지만, 읽기에는 더 불편한 상태는 아닌가

## 7. 3page PASS 조건

아래를 만족하면 `3page PASS 후보`로 본다.

- 상단 헤더 영역이 정상적으로 읽힌다
- 긴 텍스트가 박스를 넘지 않는다
- 표 내용이 읽힌다
- 옵션/가격/설명/이미지 관계가 한눈에 들어온다
- 고급 플러그인 대비 품질 저하가 크지 않다

## 8. 리뷰 단계 구분

### 개발 단계에서 완료할 것

- 상단 헤더 정렬 개선
- 긴 텍스트 줄바꿈 개선
- 표 셀/설명 가독성 개선
- 그룹과 정보 블록 배치 개선

### 사용자 검수 단계에서 할 것

- 고급 플러그인 결과와 체감 비교
- 실제로 읽고 이해 가능한지 판정
- PASS / FAIL 판단

즉, `고급 플러그인과의 최종 비교 검수`는 사용자 단계에서 진행한다.
